
# FinTrack 问题修复与使用手册重写计划

## 概览

修复5个已识别的问题，重写使用手册。涉及前端（React）、后端（Express）及文档。

## 问题1：登录错误提示闪一下就消失

**根因**：`client/src/api/client.js` 的 axios 响应拦截器在收到 401 状态码时无条件执行 `window.location.href = '/login'`（整页刷新）。登录失败时服务端返回 401，拦截器立即触发页面刷新，错误信息来不及显示。

**修复方案**：
- 修改 `client/src/api/client.js` 响应拦截器，判断是否当前已在 `/login` 页面，若是则跳过 401 重定向
- 同时检查当前 URL 路径，避免登录请求自身的 401 也触发重定向

**涉及文件**：
- `client/src/api/client.js` — 修改 401 拦截逻辑

## 问题2：刚登录进来黑屏

**根因A（主因）**：Dashboard 概览 API 字段名不匹配
- 服务端 `analytics.js` 返回：`{totalAssets, totalPrincipal, totalProfit, xirrRate, accountCount, depositCount}`（camelCase）
- 客户端 `Dashboard.jsx` 期望：`{total_assets, total_profit, xirr_return, deposit_value}`（snake_case + 不同命名）
- 结果：所有统计卡片显示 `undefined`，页面呈现空白

**根因B**：资产分布 API 数据形状不匹配
- 服务端 `/analytics/distribution` 返回：`[{grp, total}, ...]`
- Dashboard PieChart 配置：`nameKey="name"` `dataKey="balance"` → 找不到对应字段，分类显示为"??"

**修复方案**：
- 方案：修改客户端 Dashboard.jsx，使用服务端实际返回的字段名（camelCase → camelCase 适配）
  - `total_assets` → `totalAssets`
  - `total_profit` → `totalProfit`  
  - `xirr_return` → `xirrRate`
  - `deposit_value` → 新增服务端返回 `depositValue` 字段（当前未返回单独的定存价值）
- 同时在服务端 overview API 增加 `depositValue` 字段
- 修改 PieChart：`nameKey="grp"` `dataKey="total"`

**涉及文件**：
- `client/src/pages/Dashboard.jsx` — 修正字段名映射 + 修正 PieChart 配置
- `server/src/routes/analytics.js` — 在 overview 响应中增加 `depositValue` 字段

## 额外发现：账户类型前后端不一致

**问题**：前端 `TYPE_OPTIONS = ['零钱', '银行', '证券投资', '其他']`，但后端 accounts 路由验证 `isIn(['零钱','储蓄卡','证券投资','其他'])`，导致前端创建"银行"类型账户时被后端拒绝。

**修复**：将后端验证和前端统一为 `银行`（DB schema 已使用 `银行`）

**涉及文件**：
- `server/src/routes/accounts.js` — 将 `储蓄卡` 改为 `银行`（两处）
